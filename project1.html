NL2SQL using Retrieval-Augmented T5

AI-powered model that converts natural-language questions to executable SQL on the Spider dataset, enhanced with a schema-aware RAG pipeline to reduce hallucinations and improve execution accuracy.

Motivation: Writing SQL requires database expertise. This project empowers non-technical users to query relational data in plain English.

Key Contributions

  • Fine-tuned T5-base on Spider with AdamW, warm-up scheduler, gradient-accumulation, and mixed-precision training — achieved 54.96 % execution accuracy on the test set.
  • Added RAG-powered schema retrieval using LangChain, LlamaIndex, Sentence-Transformer embeddings, FAISS to supply only relevant tables/columns to the model.
  • Deployed an interactive Streamlit UI for querying; the retrieval-augmented pipeline reached 73.34 % execution accuracy on simple queries while cutting schema hallucinations.
  • Implemented checkpointing, exact-/execution-match evaluation, and schema-aware prompt formatting for better generalization.

Tools & Technologies

Python, PyTorch, Hugging Face Transformers (T5), LangChain, LlamaIndex, Sentence-Transformers, FAISS, Streamlit, SQLite, Spider dataset

Impact

Bridged the gap between natural-language questions and SQL for business users; the retrieval module improved reliability and transparency in query generation.
