Candidate Recommendation Engine

AI-powered engine that ranks resumes against a job description using semantic embeddings, keyword matching, experience parsing, and education fit, presented via a Streamlit dashboard.

Motivation: Recruiters spend hours scanning resumes manually. This project accelerates shortlisting by surfacing top candidates in seconds with transparent explanations.

Key Contributions

  • Built an embedding pipeline with Sentence-Transformers (MPNet → MiniLM fallback) and robust preprocessing/truncation.
  • Designed multi-factor ranking: 40% semantic similarity, 30% keyword coverage, 20% experience fit, 10% education match.
  • Implemented JD keyword extraction with boosted headers and multi-word phrases for precise coverage scoring.
  • Developed experience parser handling ranges like “Jan 2020–Present” and phrases like “3+ years.”
  • Matched education fields, degrees, and certifications between JD and resumes.
  • Created an Insight Engine to highlight strengths/cautions (skills overlap, seniority, metrics density) for explainability.
  • Parsed resumes (PDF/TXT) with BERT NER + regex for metadata (name/email/phone).
  • Delivered a Streamlit interface with uploads, progress bar, metrics, insights, and summary dashboards.

Tools & Technologies

Python, Streamlit, Sentence-Transformers, scikit-learn, PyTorch, HuggingFace Transformers (BERT NER/QA), fitz (PyMuPDF), Regex

Impact

Reduced candidate screening time while ensuring fairness and transparency. Clear score breakdowns and insights build trust in AI rankings for hiring teams.
